Web Application Architecture

The web application architecture describes the interactions
between applications, databases, and middleware systems on the web.
Web application architecture is a pattern of interaction between the web application components.
The way this interaction is planned out determines the resilience,
performance, and security of a future web application.
It is simply an opening a webpage. The user is able to interact with the website.
The code parsed by the web browser. This is how web application works.

In website developing, client-side and server side have different functions.
It all depends on what the web developer need to do because each tool has its purpose.
